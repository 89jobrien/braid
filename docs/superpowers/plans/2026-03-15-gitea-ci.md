# Gitea + CI Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a bare-metal Gitea instance with CI runner on jobrien-vm, push the braid repo, and verify CI runs on push.

**Architecture:** Gitea binary + SQLite on jobrien-vm (Tailscale-only Ubuntu server). act_runner in host mode executes CI jobs as the `git` user. Rust toolchain installed via rustup for the `git` user. CI workflow in `.gitea/workflows/ci.yml`.

**Tech Stack:** Gitea 1.23, act_runner 0.2, Rust (stable via rustup), systemd, SQLite

**Spec:** `docs/superpowers/specs/2026-03-15-gitea-ci-design.md`

**SSH access pattern for all remote commands:**
```bash
sshpass -p 'user-binary' ssh -o IdentitiesOnly=yes -o IdentityAgent=none dev@100.105.75.7 "COMMAND"
```

For commands requiring sudo:
```bash
sshpass -p 'user-binary' ssh -o IdentitiesOnly=yes -o IdentityAgent=none dev@100.105.75.7 "echo 'user-binary' | sudo -S COMMAND"
```

**Current machine state (verified):**
- Ubuntu 24.04, x86_64
- Git 2.43 installed
- Rust 1.75 via apt (system-wide) — NOT rustup. The `git` user needs its own rustup install.
- No `git` user exists
- No Gitea installed
- wget and curl available

## Chunk 3: CI Workflow and First Push

### Task 7: Add CI workflow to braid repo

**Target:** Local machine

**Files:**
- Create: `.gitea/workflows/ci.yml`

- [ ] **Step 1: Create workflow file**

Create `.gitea/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: ['**']
  pull_request:

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Check formatting
        run: |
          source ~/.cargo/env
          cargo fmt --all -- --check

      - name: Clippy
        run: |
          source ~/.cargo/env
          cargo clippy --workspace -- -D warnings

      - name: Test
        run: |
          source ~/.cargo/env
          cargo test --workspace
```

- [ ] **Step 2: Verify file exists**

```bash
cat .gitea/workflows/ci.yml
```

Expected: YAML with `cargo fmt`, `cargo clippy`, `cargo test` steps.

- [ ] **Step 3: Commit**

